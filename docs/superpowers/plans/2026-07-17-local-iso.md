# Local ISO (`--iso-path`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let `devboost installer` use an ISO already on disk instead of downloading it, verified against the catalog's pinned sha256.

**Architecture:** A `LocalIsoDownloader` decorator implements the existing `Downloader` Protocol: it serves one artifact (the primary install ISO) from a local path and delegates everything else (netinst, Ventoy tarball) to the real downloader. `stages.py` is not touched — all three ISO fetches already funnel through the injected `dl`. `installer()` calls `check()` **before** `build()`, so a bad ISO fails before anything is wiped.

**Tech Stack:** Python 3.12, Typer (CLI), questionary (wizard), pydantic (`MediaConfig`), pytest, `mypy --strict`, ruff.

**Spec:** `docs/superpowers/specs/2026-07-17-local-iso-design.md`

```python
def sha256_of(path: Path) -> str:
    """The lowercase hex sha256 of *path*, streamed so a 2GB ISO costs no memory.

    The single definition of "hashed" for the media pipeline: the download cache and the
    local-ISO downloader must agree byte-for-byte on what verification means.
    """
    h = hashlib.sha256()
    with path.open("rb") as fh:
        for chunk in iter(lambda: fh.read(1 << 20), b""):
            h.update(chunk)
    return h.hexdigest()
```

Replace the body of `Cache.verify` with:

```python
    def verify(self, path: Path, sha256: str) -> bool:
        if not path.exists():
            return False
        return sha256_of(path) == sha256
```

```python
class LocalIsoDownloader:
    """Serves one user-supplied ISO from disk; delegates every other artifact to *inner*.

    The user's file is used **in place**: it is never copied into the cache, so
    ``Cache.evict_stale()`` can never delete it.

    ``check()`` exists because the primary ISO is fetched from inside ``_mounted_vtoy`` —
    i.e. *after* the stick is wiped and Ventoy installed. Verifying only at fetch time would
    wipe the disk and then reject the ISO, so callers must call ``check()`` before anything
    destructive.
    """

    def __init__(self, inner: Downloader, name: str, path: Path, sha256: str) -> None:
        self._inner = inner
        self._name = name
        self._path = path
        self._sha256 = sha256
        self._checked = False

    def check(self) -> None:
        """Verify the local ISO against the catalog pin. Raises MediaError if it differs."""
        if not self._path.is_file():
            raise MediaError(
                f"--iso-path {self._path}: not a readable file"
            )
        try:
            actual = sha256_of(self._path)
        except OSError as exc:
            raise MediaError(f"--iso-path {self._path}: cannot read ({exc.strerror})") from exc
        if actual != self._sha256:
            raise MediaError(
                f"{self._path} does not match the pinned {self._name[:-4]} ISO\n"
                f"  expected {self._sha256}  (catalog.toml)\n"
                f"  got      {actual}\n"
                "This is a different release or a corrupt file. "
                "Omit --iso-path to fetch the pinned ISO."
            )
        self._checked = True

    def fetch(self, url: str, name: str, sha256: str) -> Path:
        if name != self._name:
            return self._inner.fetch(url, name, sha256)
        if sha256 != self._sha256:
            raise MediaError(
                f"{name} requested with a different sha256 than --iso-path was verified "
                f"against (expected {self._sha256}, requested {sha256})"
            )
        if not self._checked:
            self.check()
        return self._path
```
