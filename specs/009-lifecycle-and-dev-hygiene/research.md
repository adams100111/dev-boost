# Phase 0 Research: lifecycle-and-dev-hygiene

Engine-feature spec — extends `bin/devboost` + `lib/`. Decisions grounded in design doc §4/§8/§8b +
constitution (Principle V TDD binds; Principle I is satisfied — this adds engine *features*, not tools).
No new external tools, so no context7/version verification is required.

## Decisions

- **D0. New libraries, not bloated existing ones.** `lib/lifecycle.sh` (add/export/diff/update/
  self-update + lock helpers) and `lib/devhygiene.sh` (dev status/gc/down). Source-only, depend on
  `lib/log.sh`; reuse `lib/toml.sh`/`lib/profile.sh`/`lib/module.sh`/`lib/depsort.sh` for declared
  state. `bin/devboost` gains thin `cmd_*` dispatchers — existing `install/verify/list/doctor`
  untouched. All external commands PATH-stubbable.
- **D1. `devboost.lock` = deterministic sorted TSV** (`module<TAB>resolved-version`), committed,
  secret-free. Written/regenerated by `update` and `install`. Sorting by module name → clean diffs,
  reproducible across machines. Rationale: matches repo's TSV convention (servers.tsv); trivially
  diff-able; no JSON-ordering noise.
- **D2. `export` layout** = `workstation-config/exports/<UTC-timestamp>/` with per-source files:
  `dnf.txt` (`dnf repoquery --userinstalled` or `rpm -qa` fallback), `flatpak.txt` (`flatpak list
  --app --columns=application`), `mise.txt` (`mise ls`), `vscode-extensions.txt`
  (`code --list-extensions`). Read-only; a missing tool writes a `# <tool> unavailable` gap marker,
  never fails the command.
- **D3. `diff` exit codes** = 0 in sync, non-zero on drift (CI gate). Compares declared closure
  (`profile_expand full` → `depsort` → each module's `verify`) vs actual; classes: declared-but-failing
  (missing), and (best-effort) present-but-undeclared from an export. Read-only.
- **D4. `update`** proposes pins into `config/mise.toml` (seed from resolved state if absent) +
  regenerates `devboost.lock`, prints a unified-style diff, and **never** `git commit` (Principle III).
  Upstream checks are stub-injected in tests; offline → skip the check, never write an inconsistent lock.
- **D5. `add <name> [--folder]`** scaffolds `modules/<name>/module.toml` from
  `templates/module-skeleton/module.toml` (placeholders substituted: name); `--folder` also writes an
  `install.sh` skeleton sourcing `lib/log.sh`+`lib/pkg.sh`. Refuses to overwrite; rejects invalid names
  (`[a-z0-9][a-z0-9-]*`).
- **D6. `self-update`** = `git -C $DEVBOOST_ROOT pull --ff-only` then re-validate (`doctor`); on pull
  failure → named error, non-zero, no partial state.
- **D7. `dev` hygiene (design §8b).** Detection via Docker metadata:
  - AppHost/session containers carry label `com.microsoft.developer.usvc-dev.persistent` (`true`/`false`)
    and a creator-PID annotation. `dev gc` removes a container iff `persistent==false` **AND** its
    creator PID is not alive (`kill -0` fails / not in `/proc`), then `docker container prune -f`
    (exited only). Persistent or live-PID containers are never touched.
  - `dev status` lists AppHosts (project path + age + PID), per-container RAM (`docker stats --no-stream`),
    swap pressure (`/proc/meminfo` SwapFree/SwapTotal), and warns on >1 live AppHost per project path.
  - `dev down` = `ddev poweroff` + stop stale AppHosts + `docker container prune -f` + `dev gc`.
  - docker/ddev absent → graceful no-op success.
- **D8. `aspire-gc` module** = Fedora-only data module; installs `~/.config/systemd/user/
  aspire-gc.service` (oneshot `devboost dev gc`) + `aspire-gc.timer` (`OnCalendar=hourly`,
  `Persistent=true`), `systemctl --user enable --now` + `loginctl enable-linger`. Idempotent +
  verify-guarded. Mirrors Spec 8 vault-sync timer conventions.

## Testing (no real system mutation)

Extend `tests/fixtures/base/stubs.bash` (backward-compatible): `docker` (`ps`/`inspect`/`stats`/
`container prune`/`rm`/`stop` driven by knobs `STUB_DOCKER_PS`, `STUB_DOCKER_INSPECT_*`), `ddev`
(`poweroff`), `code` (`--list-extensions` → `STUB_CODE_EXTS`), `mise` (`ls`), `git` (`pull`). Reuse
existing `dnf`/`flatpak`/`systemctl --user`/`loginctl`/`rpm` stubs. Real `jq`/coreutils. Unit-test the
lib functions + bin dispatch directly; module test for `aspire-gc` via the engine.

No unresolved unknowns (all self-resolved in spec §Clarifications). Ready for Phase 1.
