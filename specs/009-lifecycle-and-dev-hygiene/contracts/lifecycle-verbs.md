# Contract: lifecycle CLI verbs (bin/devboost + lib/lifecycle.sh)

## add <name> [--folder]  (FR-001)
- valid name `^[a-z0-9][a-z0-9-]*$`; refuse if `modules/<name>` exists; create module.toml from
  templates/module-skeleton (subst __NAME__); `--folder` → install.sh sourcing lib/log.sh+lib/pkg.sh.
- prints next-step guidance. Tests: creates valid toml; refuses overwrite; rejects bad name; --folder adds install.sh.

## export  (FR-002, read-only)
- writes workstation-config/exports/<UTC-ts>/{dnf,flatpak,mise,vscode-extensions}.txt; missing tool →
  `# <tool> unavailable` line, exit 0; NO system mutation (assert no dnf/flatpak install in logs).
- Tests: snapshot files created from stub outputs; missing-tool gap; read-only.

## diff  (FR-003)
- exit 0 when every declared module verifies; exit non-zero when a declared module fails verify;
  lists drift. Read-only. Tests: in-sync→0; force a module verify-fail→non-zero + names it.

## update [--profile X]  (FR-004, FR-005)
- seeds config/mise.toml if absent; writes proposed pins; regenerates devboost.lock; prints diff;
  asserts NO `git commit` ever invoked (grep STUB_GIT_LOG). Tests: lock regenerated sorted; no commit.

## self-update  (FR-006)
- `git -C $DEVBOOST_ROOT pull --ff-only` then doctor; pull-fail (STUB_GIT_PULL_FAIL=1) → non-zero + named.
- Tests: pull invoked then revalidate; failure path named.

## devboost.lock  (FR-005)
- sorted TSV `module<TAB>version`; deterministic; secret-free; regenerated by update + install.
